# 药店AI系统业务流程图

## 一、语音实时识别服务

### 1. 用户发起请求
- **接口**: `POST /api/tingwu/realtime/start`
- **参数**: 语言、音频格式、采样率

### 2. 控制器接收请求
- **组件**: `TingwuRealtimeController`
- **动作**: 接收并解析请求参数

### 3. 创建听悟任务
- **组件**: `TingwuService`
- **动作**: 调用阿里云听悟API创建实时任务
- **返回**: TaskId、WebSocket连接地址

### 4. 建立WebSocket连接
- **组件**: `TingwuWebSocketService`
- **动作**: 与阿里云NLS服务建立WebSocket连接

### 5. 启动麦克风采集
- **音频格式**: 16kHz / 16bit / 单声道 / PCM
- **动作**: 开始从麦克风采集音频数据

### 6. 音频数据传输
- **数据块**: 每次发送 3200 字节（约100ms音频）
- **方向**: 本地 → 阿里云 NLS 服务

### 7. 云端语音识别
- **服务**: 阿里云 NLS 实时语音识别
- **动作**: 将音频流转为文字

### 8. 返回转写结果
- **回调**: `onSentenceEnd`
- **输出**: 转写文本

## 二、语义分析服务

### 1. 接收转写结果
- **触发**: 语音识别完成回调
- **输入**: 转写文本

### 2. 对话判断
- **条件**: 是否收集到完整对话轮次？
- **否**: 继续监听等待
- **是**: 进入下一步分析

### 3. 上下文管理
- **组件**: `ContextManager`
- **动作**: 将对话添加到上下文队列

### 4. 维护历史对话
- **策略**: 保留最近 3 轮对话
- **结构**: FIFO 队列

### 5. 构建Prompt
- **动作**: 组合历史对话 + 当前输入
- **角色设定**: 药店销售助手

### 6. 调用大模型API
- **组件**: `LlmService.streamAnalyze`
- **协议**: HTTP POST (OpenAI兼容格式)
- **模型**: DeepSeek / 其他大模型

### 7. 流式返回分析结果
- **格式**: SSE 流式传输
- **内容**: 逐字返回分析结果

### 8. 解析与输出
- **组件**: `ConsoleOutputService`
- **输出字段**:
  - 顾客需求（如：退烧药、感冒药）
  - 推荐药品类别
  - 紧急程度
  - 置信度
  - 建议追问

### 9. 循环判断
- **条件**: 是否有新对话？
- **是**: 返回步骤3继续分析
- **否**: 等待下一轮对话

## 数据流向图

```
[用户] → [Controller] → [TingwuService] → [阿里云听悟]
                                              ↓
[输出结果] ← [ConsoleOutput] ← [LlmService] ← [NLS转写结果]
                   ↑
            [ContextManager]
```

## 技术栈

| 模块 | 技术 |
|------|------|
| 语音识别 | 阿里云 NLS SDK |
| 实时传输 | WebSocket |
| 音频采集 | Java Sound API |
| 大模型 | DeepSeek API (OpenAI兼容) |
| 上下文管理 | 内存队列 (LinkedList) |
